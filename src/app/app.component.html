Real-time Medical Data Monitoring System

This system is designed to simulate the communication between a medical device (client) and a server application, storing and visualizing vital sign data in real-time. The system comprises three main components:
Client (Medical Device Simulator): Simulates a medical device generating vital sign data and sending it to the server.
Server: Listens for incoming connections from client devices, stores the received data in a Redis database.
GUI (Data Visualization): Provides a user-friendly interface to visualize and search for patient vital sign data stored in Redis.
